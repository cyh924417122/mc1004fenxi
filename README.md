## 🔧 我能干什么

本项目提供以下功能：

### 1. 计算 OEM 哈希与生成注册载荷

**脚本**：`oem_hash_payload.py`

根据产品 ID 计算 OEM 哈希值，并生成 16 字符的注册载荷（前 12 位数字 + 3 位字母 OEM 编码 + 1 位尾部字符）。

```bash
python3 oem_hash_payload.py
```

**示例输出**：
```
产品ID: 15044657
OEM哈希: 5074
编码字符: KQR
注册载荷: 231251190900KQRX
```

---

### 2. 生成 20 位注册码

**脚本**：`generate_checksum.py`

根据产品 ID 和注册载荷，生成经过多层加密（伪随机映射 + 数据旋转 + 双重校验）的 20 位数字注册码。

```bash
python3 generate_checksum.py
```

**示例输出**：
```
产品ID: 15044657
注册载荷: 231251190900KQRX
生成的20位校验码: 29592544820702887880
```

---

### 3. 解码并验证注册码

**脚本**：`decode_registration.py`

对 20 位注册码进行逆向解码，还原出产品码、日期、尾数等字段，并通过双重校验（校验和 + 哈希）验证注册码合法性。

```bash
python3 decode_registration.py
```

**示例输出**：
```
验证之前生成的校验码结果:
输入校验码: 29592544820702887880
状态码: 0
产品码: 15044657
日期: 512
尾数: 20

校验码验证成功！
```

**状态码说明**：

| 状态码 | 含义 |
|:------:|------|
| `0` | 校验成功，所有验证通过 |
| `1` | 校验和失败，前 19 位数字之和对 10 取模不等于第 20 位 |
| `2` | 内部校验失败，哈希值与提取的校验值不匹配 |

---

### 4. OEM 哈希计算器（图形界面）

**脚本**：`oem_hash_calculator.py`

提供可视化图形界面，方便交互式输入产品 ID 并实时查看 OEM 哈希值与编码字符。

```bash
python3 oem_hash_calculator.py
```

> **注意**：需要系统支持 `tkinter` 图形库。

## 🔑 注册码系统原理

注册码系统采用多层安全机制：

1. **OEM 哈希**：基于 23 进制线性同余算法，将产品 ID 映射为 3 个字母字符。
2. **数据旋转**：根据校验和的末位数字对 19 位数据进行循环移位，混淆数据排列。
3. **伪随机映射**：使用线性同余 PRNG（模数 `0x14DB3`，乘数 `997`）生成 0–9 的随机排列映射表，将数字序号转换为最终输出值。
4. **双重校验**：同时验证前 19 位数字之和（校验和）与数据哈希值，确保注册码完整性。

详细算法说明请参阅：[注册码系统实施文档.md](./注册码系统实施文档.md)

## 🚀 快速开始

### 环境要求

- Python 3.6 及以上版本
- 如需使用图形界面工具，需安装 `tkinter`（通常随 Python 标准库附带）

### 运行示例

```bash
# 生成注册载荷
python3 oem_hash_payload.py

# 生成 20 位注册码
python3 generate_checksum.py

# 验证注册码
python3 decode_registration.py
```
